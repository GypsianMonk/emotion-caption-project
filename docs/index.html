// Architecture

Two Parallel Pipelines

Each model is independently trained, optimized, and exportable — then unified in a single real-time inference loop.

🧠

Emotion Recognition CNN

FER-2013 · 35,887 images · 7 classes

→Input Frame (48×48 grayscale)
→Conv32 → BN → ReLU → MaxPool
→Conv64 → BN → ReLU → MaxPool
→Conv128 → BN → ReLU → MaxPool
→Conv256 + Residual Block
→GlobalAvgPool → Dense(512)
→Softmax (7 emotions)
📷

InceptionV3 + LSTM Captioning

MS-COCO · 80K images · Beam Search

→Image (299×299 RGB)
→InceptionV3 (frozen, ImageNet)
→Feature Vector (2048,)
→Dense Projection (512,)
→Bahdanau Attention
→LSTM Decoder (512 units)
→Beam Search (width=3)
// Features

Production-Grade Design

Built to the same standards as a deployed ML service — not a notebook demo.

⚡

Real-Time Pipeline

Emotion detection runs every frame in the main thread. Caption generation runs in a background thread — video never blocks.

🎯

Beam Search Decoder

Length-normalized beam search (width 1–5) with Bahdanau attention. Greedy fallback for low-latency scenarios.

📦

TFLite Export

INT8 quantized emotion model for edge deployment. ~3–4× faster on CPU with less than 1% accuracy drop.

⚖️

Class Balancing

FER-2013 is heavily skewed toward Happy/Neutral. Sklearn balanced class weights fix training bias automatically.

🗄️

Feature Caching

InceptionV3 features extracted once and cached per image with MD5 keys. Cuts captioning training time by ~10×.

🐳

Docker Ready

GPU-enabled Dockerfile with health checks, multi-worker uvicorn, and environment variable model loading.
